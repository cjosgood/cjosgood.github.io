Full Stack Application: Client Design

April 16, 2021

Update for Phase 3

For Phase 3 of the full stack chat application, my team members and I are explaining what the client side design is going to be like and the different views that the user will have. Below are some visions we have come up with.

Client Logic:

JavaScript’s logic and data processing responsibilities will include the following features:

  • User authentication: JS will allow the user to log in with a username and password as well as have an email connected to the account. Similar to Lab 12, will will use Passport, EJS, Bcrypt in order to accomplish user authentication.
  • Messaging: JS will process the data from users participating in conversations.

Server Fetches:

All expected server requests will include the following:

  • Register: POST user information into database to be fetched at sign-in
  • Log-in: Use client’s input to GET account from database, if credentials are correct.
  • Friend requests: POST a friend request that is sent to the target account. They can accept or deny the request.
  • Chat: POST and GET messages, dependent on if the client sent a message or received a message from another user.

Tooling and Libraries:

  • CSS
  • JS
  • JSON
  • HTML
  • React
  • Bootstrap
  • Bcrypt
  • EJS
  • Passport
